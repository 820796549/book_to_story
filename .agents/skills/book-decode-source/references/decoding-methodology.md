# 原文拆解方法论

## 拆解目标

原文拆解不是写读后感，也不是做摘要。

拆解的目标是：**为栏目文章生产提供事实、判断和依据。**

拆解完成后，写手应该能回答：
- 这篇原文在解决什么问题？
- 作者依据什么做判断？
- 论证如何一步步展开？
- 哪些内容可以用于栏目文章？
- 哪些内容不能过度发挥？

## 拆解流程

### 第一步：通读原文

通读全文，标记以下内容：
- 核心问题（原文要回答什么）。
- 关键事实（时间、地点、人物、事件、数据）。
- 作者判断（核心论点）。
- 论证步骤（如何展开）。
- 关键概念（重要术语）。

### 第二步：提炼核心问题

不要只写"本文介绍了什么"。

必须拆出三层：
- 表层问题：文章表面上在讨论什么。
- 深层问题：文章实际在解决什么。
- 真正问题：为什么这篇文章必须在此时出现。

示例（中国社会各阶级的分析）：
- 表层问题：中国社会有哪些阶级？
- 深层问题：各阶级对革命什么态度？
- 真正问题：谁是敌人？谁是朋友？过去为什么失败？

### 第三步：拆推理链

推理链必须按链条写，不能写成零散要点。

标准格式：
```
现实处境 -> 核心问题 -> 判断依据 -> 分类/分析 -> 结论 -> 行动含义
```

### 第四步：提取可引用原文

只摘短句或关键句。每条引用标注用途：
- 原文：
- 用途：
- 可用于哪个栏目：
- 注意边界：

### 第五步：标注不可过度发挥的边界

明确标注：
- 哪些是原文事实。
- 哪些是作者判断。
- 哪些是现代人不能硬套的。
- 哪些是写作时不能歪曲的。

## 输出模板

```
# 原文拆解：篇名

## 1. 原文信息
## 2. 写作背景
## 3. 核心问题（表层/深层/真正）
## 4. 关键事实
## 5. 作者判断
## 6. 论证步骤
## 7. 关键概念
## 8. 推理链
## 9. 可提炼模型
## 10. 可引用原文
## 11. 可迁移点
## 12. 不可过度发挥的边界
```
