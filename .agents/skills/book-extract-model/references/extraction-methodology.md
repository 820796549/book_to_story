# 模型提炼方法论

## 提炼目标

模型提炼不是编口号，而是从原文结构中提取可复用的判断方式。

## 模型类型

可以提炼的模型类型：

| 类型 | 说明 | 示例 |
|---|---|---|
| 问题识别模型 | 如何识别真正的问题 | 敌友识别模型 |
| 局势判断模型 | 如何判断局势中的力量 | 力量判断模型 |
| 分类模型 | 如何对复杂对象分类 | 阶级分析模型 |
| 决策模型 | 如何做决策 | 利益锚定模型 |
| 行动模型 | 如何行动 | 统一战线模型 |
| 认知纠偏模型 | 如何纠正错误认知 | 真相挖掘漏斗 |

## 提炼标准

每个模型必须满足：

1. 来自原文结构，不是凭空编造。
2. 有原文依据（引用原文句子）。
3. 有清楚的结构（变量+判断顺序）。
4. 有使用场景。
5. 有失效边界。

## 模型结构格式

```
模型名称：
解决问题：
核心变量：
判断顺序：
行动含义：
适用场景：
失效边界：
```

## 推理链格式

```
现象 -> 问题 -> 变量 -> 判断 -> 结论 -> 行动含义
```

## 原文依据

每个模型必须说明来自原文哪部分：

```
原文依据：
对应逻辑：
可引用短句：
```

如果模型主要来自现代发挥，必须标注：非原文直接模型，属于现代迁移解释，需审查。

## 禁止

- 不把口号当模型。
- 不为了押韵或整齐编模型。
- 不脱离原文强行升华。
- 不把复杂判断压成一句空话。
