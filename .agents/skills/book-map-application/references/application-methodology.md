# 现实迁移方法论

## 迁移目标

现实迁移不是"古今都一样"的硬套，而是找到结构相似的同构关系。

## 迁移标准

每个迁移必须满足三个条件：

1. 现代处境和原文问题有结构相似性。
2. 迁移后能帮助读者重新判断。
3. 不把历史结论硬套到现代问题。

## 迁移链格式

```
原文问题 -> 原文模型 -> 现代同构处境 -> 读者错误反应 -> 新判断方式 -> 可执行启发
```

## 现代痛点要求

痛点必须具体。

不合格：现代人很焦虑。
合格：很多人不是不努力，而是把合作对象看错了：他拼命靠近看起来最强的人，却忽略了真正和自己利益相连的人。

## 错误认知要求

必须拆出读者原本怎么想错。

格式：
```
读者以为：
真实问题：
错误后果：
新判断：
```

## 迁移边界

必须标注：
- 哪些是合理迁移（同构关系明确）。
- 哪些是过度发挥（没有原文依据）。
- 哪些是绝对不能写的。

## 禁止

- 不写鸡汤。
- 不把历史结论直接套给现代人。
- 不替用户做具体人生决策。
- 不为了爽感制造极端判断。
- 不脱离原文模型自由发挥。
- 不把复杂现实说成一句万能公式。
