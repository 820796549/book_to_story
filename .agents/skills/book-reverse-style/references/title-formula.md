# 标题公式

## 底层解码篇标题

### 公式

```
底层解码篇丨[核心问题/悬念/判断]
```

### 类型

| 类型 | 示例 | 适用场景 |
|---|---|---|
| 为什么类 | 为什么过去的斗争总是失败？ | 原文解决了一个历史困惑 |
| 反问类 | 为什么只有中国能长出红军？ | 原文有独特的条件分析 |
| 悬念类 | 当所有人都怀疑红旗能打多久时... | 原文在悲观时刻给出判断 |
| 判断类 | 不再做留声机 | 原文给出了一个明确转向 |
| 概念类 | 井冈山斗争的战略辩证法 | 原文有系统的战略分析 |
| 如何类 | 破局混沌：古田会议如何重塑组织认知 | 原文解决了组织问题 |

### 规则

1. 必须让读者产生好奇。
2. 不能是干巴巴的"读《XXX》有感"。
3. 不能是纯概念堆砌。
4. 不能太长，不超过 30 字（不含栏目前缀）。

## 认知破局篇标题

### 公式

```
认知破局篇丨[现代痛点/认知翻转/行动启发]
```

### 类型

| 类型 | 示例 | 适用场景 |
|---|---|---|
| 痛点类 | 你总觉得自己力量不够？ | 直接命中读者困境 |
| 翻案类 | 你不是不努力，你是把局势看反了 | 推翻读者错误认知 |
| 方法类 | 只有调查才有发言权 | 给出可执行方法 |
| 概念类 | 战略生态系统 | 建立系统框架 |

### 规则

1. 必须从现代读者痛点进入。
2. 不能是"读《XXX》的启示"。
3. 不能是纯历史标题。
4. 必须让读者觉得"这说的是我"。
