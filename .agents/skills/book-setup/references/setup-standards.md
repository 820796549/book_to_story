# 项目初始化规范

## 初始化目标

把一个空目录或已有书籍目录初始化成可定位、可反推、可拆原文、可写栏目文章、可审查、可沉淀方法库的项目。

## 标准目录

```
00-项目定义/
01-样文反推/反推记录/
01-样文反推/深反推标杆/
02-原文拆解/
03-模型提炼/现实迁移/
04-栏目协议/
05-文章生产/草稿/
05-文章生产/成稿/
06-质量审查/审查报告/
07-方法库/
08-项目打包/
追踪/
```

## 初始化文件

只在文件不存在时创建：

- 00-项目定义/书籍定位.md
- 00-项目定义/目标用户.md
- 00-项目定义/栏目体系.md
- 追踪/上下文.md
- 追踪/session-log.md

## 部署脚本

在项目根目录运行：

```bash
python scripts/book-setup.py
```

该脚本会自动创建标准目录、初始化文件和部署标记。

## Codex 配置

部署后需要：
1. 信任当前项目 .codex 配置层。
2. 在 /hooks 中 review/trust hooks。
3. 新开 Codex 会话让 custom agents 生效。

## 禁止

- 初始化时直接写正文。
- 覆盖用户已有内容。
- 把所有书默认设成毛选。
- 把所有书默认设成底层解码篇/认知破局篇。
