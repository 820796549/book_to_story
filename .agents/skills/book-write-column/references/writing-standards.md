# 栏目文章写作规范

## 写作前置

写正文前必须确认存在：
- 00-项目定义/书籍定位.md
- 00-项目定义/栏目体系.md
- 04-栏目协议/对应栏目协议.md
- 01-样文反推/对应样文反推.md
- 02-原文拆解/对应篇章.md
- 03-模型提炼/对应模型.md

认知破局类文章还需要：03-模型提炼/现实迁移/对应迁移.md

## 底层解码篇结构

```
标题
-> 历史场景（时间+地点+压力）
-> 历史棋盘（外部矛盾+内部矛盾+主流误判）
-> 核心问题（用原文第一句锚定）
-> 推理链（判断标准+逐层分析）
-> 破局点（关键转向）
-> 历史验证（后续历史证明）
-> 收束（核心判断）
```

固定要素：历史处境、当时误判、推理链、破局点。

## 认知破局篇结构

```
标题
-> 现代痛点（具体困境）
-> 错误认知（读者怎么想错）
-> 引入毛选模型（原文判断方式）
-> 历史材料支撑（原文推导）
-> 现实迁移（现代同构处境）
-> 认知转向（新判断方式）
-> 爽点句强化（关键记忆点）
-> 收束（可执行启发）
```

固定要素：现代痛点、错误认知、毛选模型、现实迁移、爽点句。

## 文章要求

必须同时满足：
1. 有明确读者问题。
2. 有原文依据。
3. 有清楚推理链。
4. 有栏目边界。
5. 有读感和传播性。
6. 有可记住的判断句。

## 标题要求

- 底层解码篇：底层解码篇丨[核心问题/悬念/判断]
- 认知破局篇：认知破局篇丨[现代痛点/认知翻转/行动启发]

## 原文依据要求

- 可以短引用，不能大段搬运。
- 每个关键判断都要有原文拆解支撑。
- 引用后解释"这句话为什么重要"。

## 爽点句要求

- 每篇至少 3 类可记忆判断：点破误判、提炼模型、迁移现实。
- 爽点句必须来自推理链，不能无依据制造狠话。

## 禁止

- 不缺材料硬写。
- 不写成原文摘要。
- 不写成鸡汤。
- 不混淆栏目。
- 不复制竞品表达。
- 不脱离原文过度发挥。
