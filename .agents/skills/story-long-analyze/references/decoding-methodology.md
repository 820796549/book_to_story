# 拆书方法论：原文解码

## 核心原则

拆书不是读书笔记。拆书的目的是从原文中提取可迁移的认知模型和判断框架。

## 解码三问

每篇原文必须回答三个递进问题：

1. **他做了什么**（事实层）
2. **他为什么这么做**（逻辑层——信息约束下的决策推演）
3. **我该如何迁移这套思维**（应用层——映射到当代场景）

## 解码框架

### 1. 处境分析
- 当时面临什么局面？
- 有哪些约束条件（信息不全、资源匮乏、强敌环伺）？
- 关键变量是什么？

### 2. 问题识别
- 表层问题是什么？
- 深层问题是什么？
- 真正的问题是什么？

### 3. 判断依据
- 作者依据什么做判断？
- 数据来源是什么？
- 推理前提是什么？

### 4. 推理链
- 现实处境 -> 核心问题 -> 判断依据 -> 分类/分析 -> 结论 -> 行动含义

### 5. 模型提取
- 这个案例中隐含了什么判断模型？
- 模型包含哪些变量？
- 模型的适用边界在哪里？

### 6. 迁移映射
- 现代场景中哪些处境和原文同构？
- 原文模型如何适配现代场景？
- 迁移的边界在哪里？

## 常见错误

- 只总结内容，不拆推理链
- 只摘金句，不提炼模型
- 把现代应用强行塞进原文
- 忽略原文的约束条件
- 为了文章爽感提前发挥
