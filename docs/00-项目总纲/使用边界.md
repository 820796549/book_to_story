# 使用边界

## 1. 总边界

本项目只解决一个核心问题：

```text
如何把有用书稳定拆成高质量文章。
```

所有使用方式都必须围绕这个问题。

如果一个动作不能帮助“写出更好的文章”或“让文章生产更稳定”，就不是第一阶段重点。

## 2. 可以做什么

本项目可以做：

1. 为一本书做项目定位。
2. 为一本书设计栏目体系。
3. 反推高质量样文写法。
4. 从原文中拆出事实、问题、推理链、模型和迁移点。
5. 写用户可见的长文文章。
6. 审查文章是否有原文依据、是否好读、是否有现实价值。
7. 把成熟写法沉淀成 skill、模板、references 和 hooks。
8. 用样板项目训练出可复用流程。

## 3. 暂时不做什么

第一阶段暂时不做：

1. 不做全书知识库产品包装。
2. 不做前端网站。
3. 不做自动批量发布。
4. 不做多书并行。
5. 不做课程体系。
6. 不做播客体系。
7. 不做销售页。
8. 不做学术考据。
9. 不做无限泛化。

这些不是永远不做，而是不能抢在“文章质量稳定”之前做。

## 4. 书籍边界

不是所有书都适合进入本项目。

适合的书通常具备至少一种价值：

- 能解释现实问题。
- 能提供判断方法。
- 能提供行动策略。
- 能提供精神支撑。
- 能提供表达素材。
- 能提供可迁移模型。

不适合第一阶段进入的书：

- 只适合娱乐消遣的书。
- 主要价值在资料检索的书。
- 需要大量版本考据才能使用的书。
- 无法形成用户问题入口的书。
- 没有明确文章读者的书。

## 5. 栏目边界

栏目不能先验固定。

正确流程是：

```text
先判断书的核心价值
再判断用户要解决什么问题
最后设计栏目
```

错误流程是：

```text
所有书都套用底层解码篇和认知破局篇
```

《毛选》可以使用这两个栏目，是因为它自身适合：

- 底层解码篇：拆历史处境、误判、推理链、破局点。
- 认知破局篇：拆现代痛点、错误认知、毛选模型、现实迁移。

换一本书，必须重新判断。

## 6. 竞品边界

竞品只负责提供写法样本。

可以从竞品学习：

- 标题结构。
- 开头钩子。
- 痛点进入方式。
- 历史材料组织方式。
- 推理链展开方式。
- 爽点句写法。
- 现实迁移方式。
- 结尾收束方式。

不能从竞品继承：

- 项目定位。
- 所有栏目名称。
- 所有内容观点。
- 长期生产边界。
- 直接照搬表达。

竞品反推的最终目的，是让系统形成自己的写法库。

## 7. oh-story 边界

oh-story 是工程底座，不是内容方向。

可以学习：

- skill 路由。
- agent 分工。
- hooks 守卫。
- references 按需加载。
- 目录结构。
- 进度追踪。
- 审查机制。

不能照搬：

- 小说正文生产逻辑。
- 人设/剧情/伏笔优先级。
- 网文章节推进方式。
- 只为小说服务的文件结构。

本项目必须把 oh-story 改造成拆书写作系统，而不是给小说系统换名字。

## 8. AI 使用边界

AI 不能直接跳到正文。

写正文前必须先具备：

1. 书籍定位。
2. 栏目协议。
3. 原文拆解。
4. 样文写法规则。
5. 当前文章目标。
6. 质量验收标准。

如果缺少以上内容，AI 应该回到前置步骤，而不是继续写正文。

## 9. 毛选样板边界

《毛选》第一阶段只负责验证系统能力。

验证目标是：

```text
能否稳定生产接近竞品质量的底层解码篇和认知破局篇。
```

《毛选》样板不负责证明系统已经适配所有书。

只有当《毛选》样板跑通后，才能进入第二本书验证。

## 10. 失败信号

出现以下情况，说明项目正在跑偏：

1. 文件越来越多，但没有试写文章。
2. 一直讨论通用系统，不解决毛选文章质量。
3. 一直复刻竞品，却没有自己的写法库。
4. 一直研究 oh-story，却没有改成拆书流程。
5. 所有书都套毛选栏目。
6. 直接让 AI 写正文，不做原文拆解。
7. 审查只说好不好，不指出结构问题。
8. 每次任务都改变主线。

出现这些信号时，必须回到一句话目标：

```text
拆有用书，写高质量文章，并让这个过程稳定复用。
```
