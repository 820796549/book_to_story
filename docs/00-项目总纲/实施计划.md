# 拆有用书写文章生产系统：实施计划

## 1. 项目目标

本项目的目标不是做普通读书笔记，也不是先做大而全知识库。

本项目的目标是：

```text
把有用书拆成高质量文章，
并把这个过程工程化，做到稳定复用。
```

这里的“有用书”指的是能给用户提供现实理解、判断方法、行动启发、表达素材或精神支撑的书。

第一阶段不追求适配所有书。第一阶段先用《毛选》跑通样板，因为《毛选》最适合验证：

- 如何从原文中拆出问题意识。
- 如何还原具体处境下的判断过程。
- 如何把历史材料转成现代人的现实启发。
- 如何写出有读感、有爽点、有传播性的长文。

## 2. 四个固定关系

本项目后续不再改这四个关系。

### 2.1 目标

目标是拆有用书写文章。

所有工程、模板、skill、agent、hook 都服务这个目标。

### 2.2 项目

项目只是让“拆书写文章”这个过程稳定复用。

项目不等于堆文件，不等于知识库目录，不等于让 AI 批量生成内容。

### 2.3 竞品

竞品只用来学习写法。

竞品不决定项目边界，不决定所有书都必须写成同样栏目，也不作为长期依赖。

竞品反推完成后，沉淀为自己的写法库、栏目协议和审查标准。

### 2.4 oh-story

oh-story 只作为工程底座。

要学习的是它的工程化方式：

- skill 路由。
- agent 分工。
- hook 守卫。
- references 按需加载。
- 项目结构标准化。
- 生产流程可追踪。

不是把拆书项目做成网文项目，也不是照搬小说生产逻辑。

## 3. 第一阶段边界

第一阶段只做一件事：

```text
把 oh-story 的工程底座改造成“拆有用书写文章”的生产系统，
并用《毛选》作为第一个样板项目跑通。
```

第一阶段不做：

- 不先做庄子。
- 不先做小岛经济学。
- 不先做苏东坡。
- 不先追求全书通用。
- 不先批量生成几十篇文章。
- 不先做复杂前端。
- 不先做知识库产品包装。

这些都等《毛选》样板跑通后再做。

## 4. 核心流程

系统固定流程如下：

```text
书籍定位
-> 栏目设计
-> 样文反推
-> 写法库沉淀
-> 原文拆解
-> 单篇试写
-> 对标审查
-> 修正协议
-> 小批量生产
-> 项目沉淀
```

任何一本书进入系统，都必须先走“书籍定位”和“栏目设计”。

不能直接套用《毛选》的“底层解码篇”和“认知破局篇”。

## 5. 毛选样板的栏目边界

《毛选》第一阶段固定两个栏目。

### 5.1 底层解码篇

底层解码篇负责拆历史处境和决策算法。

固定关注：

```text
历史处境
-> 当时误判
-> 教员推理链
-> 破局点
```

它不是现代鸡汤，也不是读后感。

### 5.2 认知破局篇

认知破局篇负责拆现代痛点和现实迁移。

固定关注：

```text
现代痛点
-> 错误认知
-> 毛选模型
-> 现实迁移
-> 爽点句
```

它不是单纯讲历史，也不是机械总结原文。

## 7. 质量标准

一篇文章合格，不看文件是否生成，而看四件事：

1. 是否有明确用户问题。
2. 是否有原文依据。
3. 是否有清楚推理链。
4. 是否有可读性和传播性。

一套流程合格，不看目录是否复杂，而看四件事：

1. 是否能防止方向跑偏。
2. 是否能减少临场提示词碰运气。
3. 是否能把竞品写法转成自己的方法库。
4. 是否能让下一篇文章比上一篇更稳定。
