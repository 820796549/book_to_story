# 项目定位

## 1. 一句话定位

本项目是一个“拆有用书写文章”的工程化生产系统。

它把一本有现实价值的书，从原文材料转成用户愿意读、读得懂、用得上、能传播的文章。

## 2. 项目要解决的问题

用户真正要的不是“拆书文件夹”，而是高质量文章。

过去的问题是：

- 看到竞品文章质量高，但不知道它到底好在哪里。
- 直接让 AI 写，出来的内容像总结、像鸡汤、像空话。
- 每次换一本书，栏目、写法、质量标准都重新混乱。
- 项目文件越来越多，但文章质量没有稳定提升。
- AI 容易跳过定位、样文反推、栏目协议，直接生成正文。

本项目要解决的就是这件事：

```text
把“凭感觉写一篇好文章”
变成“按流程稳定生产高质量文章”。
```

## 3. 它是什么

本项目是：

1. 拆书写作系统。
2. 高质量文章生产系统。
3. 样文反推和写法沉淀系统。
4. 基于 oh-story 工程底座改造的 skill/agent/hook 系统。
5. 可以随着书籍项目逐步复用的内容生产工作流。

它服务的核心动作是：

```text
选一本有用书
-> 判断这本书的用户价值
-> 设计适合这本书的栏目
-> 学习高质量样文写法
-> 拆原文
-> 写文章
-> 审查文章
-> 沉淀方法
```

## 4. 它不是什么

本项目不是：

1. 普通读书笔记系统。
2. 全文摘要系统。
3. 金句摘抄系统。
4. 学术考据系统。
5. 批量洗稿系统。
6. 只服务《毛选》的单书项目。
7. 一上来适配所有书的泛化系统。
8. 单纯复刻竞品文章的仿写工具。
9. 单纯把 oh-story 改几个名字的换皮项目。

尤其要明确：

```text
竞品是写法老师，不是项目主人。
oh-story 是工程底座，不是内容方向。
毛选是第一套样板，不是全部边界。
```

## 5. 第一阶段为什么先做毛选

第一阶段先做《毛选》，不是因为项目只做毛选。

原因是《毛选》有三个优势：

1. 它天然有强处境、强问题、强决策、强破局。
2. 用户已经有竞品样文，可以反推高质量写法。
3. 它能快速检验系统是否真的能写出有读感的文章。

所以《毛选》的角色是：

```text
样板工程
```

不是：

```text
最终边界
```

如果《毛选》都跑不通，说明系统没有能力稳定写文章。

如果《毛选》跑通，后面再进入第二本书，重新做书籍定位和栏目设计。

## 6. 栏目原则

栏目必须由书本身决定。

不能把任何书都套成：

```text
底层解码篇
认知破局篇
```

这两个栏目只适合《毛选》第一阶段样板。

原因是《毛选》的核心价值可以被拆成两条线：

```text
历史处境中的决策算法
现代处境中的认知破局
```

换成《庄子》，栏目可能变成：

```text
寓言解码
心智松绑
处世困局
概念辨析
```

换成《小岛经济学》，栏目可能变成：

```text
经济寓言解码
机制模型拆解
现实经济现象映射
普通人决策启发
```

因此通用的是流程，不是栏目名称。

## 7. 竞品的作用

竞品只做三件事：

1. 反推它为什么好读。
2. 反推它如何制造痛点、爽点和转折。
3. 把可复用写法沉淀成自己的写法库。

竞品不做三件事：

1. 不决定所有书的栏目。
2. 不决定项目最终形态。
3. 不作为长期复制对象。

竞品反推的终点不是“继续看竞品”，而是形成自己的：

```text
标题公式
开头钩子
痛点入口
历史开场
推理链模板
爽点句类型
现实迁移方法
结尾套路
禁用写法
```

## 8. oh-story 的作用

oh-story 的价值不在提示词多，而在工程化。

本项目学习它的这些做法：

- 用入口 skill 做路由。
- 用专业 skill 承担不同任务。
- 用 agent 做角色分工。
- 用 hooks 阻止错误流程。
- 用 references 存放方法论，不预占上下文。
- 用标准目录承载长期项目。
- 用审查环节控制质量。

本项目不照搬它的内容逻辑。

小说系统关心的是：

```text
人设、剧情、伏笔、节奏、爽点、章节推进
```

拆书写作系统关心的是：

```text
书籍价值、用户问题、原文依据、模型提取、现实迁移、文章表达
```

## 9. 成功标准

项目第一阶段成功，必须同时满足：

1. 能清楚说明一本书为什么值得拆。
2. 能为这本书设计适合它的栏目。
3. 能把竞品样文反推出可复用写法。
4. 能基于原文写出一篇合格文章。
5. 能通过审查发现文章弱点。
6. 能把审查结果反馈到模板和协议。
7. 能像 oh-story 一样被 skill 化调用。

如果只是生成很多文件，不算成功。

如果文章质量仍然靠临场提示词，不算成功。

如果所有书都被写成毛选风格，不算成功。

## 10. 当前固定方向

当前方向固定为：

```text
用 oh-story 的工程底座，
学习竞品的文章写法，
先跑通《毛选》样板，
最终形成“拆有用书写文章”的稳定生产系统。
```
