# hook 守卫设计

## 1. 设计目标

hook 的作用不是增加形式，而是阻止项目跑偏。

本项目最容易出问题的地方是：

```text
还没有定位，就开始写正文。
还没有样文反推，就要求写得像竞品。
还没有原文拆解，就开始发挥。
还没有审查，就批量生产。
```

hook 要守住这些关键流程。

## 2. 守卫原则

hook 只守机械规则，不替代内容判断。

它应该检查：

- 文件是否存在。
- 目录是否完整。
- 当前动作是否越过前置步骤。
- 是否存在明显禁止行为。

它不应该判断：

- 文章是否精彩。
- 观点是否深刻。
- 表达是否高级。

这些由 review skill 或 reviewer agent 处理。

## 3. session-start

### 3.1 触发时机

每次打开项目会话时触发。

### 3.2 功能

显示当前项目状态。

建议输出：

```text
当前书籍：
当前阶段：
项目定位：已完成/缺失
栏目体系：已完成/缺失
样文反推：已完成/缺失
原文拆解：已完成/缺失
文章成稿：数量
质量审查：数量
下一建议动作：
```

### 3.3 目的

让新会话不丢上下文，避免 AI 重新发明流程。

## 4. session-end

### 4.1 触发时机

每次会话结束时触发。

### 4.2 功能

记录会话日志。

建议写入：

```text
追踪/session-log.md
```

记录内容：

- 本次完成了什么。
- 修改了哪些文件。
- 当前阶段。
- 下次应继续什么。
- 是否存在阻塞。

## 5. pre-write-column

### 5.1 触发时机

当系统准备写入：

```text
05-文章生产/成稿/
```

### 5.2 阻断条件

如果缺少以下文件或目录，阻断写入：

```text
00-项目定义/书籍定位.md
00-项目定义/栏目体系.md
04-栏目协议/
02-原文拆解/
```

如果文章要求“像竞品”，还必须存在：

```text
01-样文反推/
```

### 5.3 提示文案

```text
禁止直接写正文：当前项目缺少前置材料。
请先完成书籍定位、栏目协议、样文反推或原文拆解。
```

## 6. guard-column-protocol

### 6.1 触发时机

当用户要求写某个栏目文章时触发。

### 6.2 检查内容

检查是否存在对应栏目协议。

例如写《毛选》底层解码篇，必须存在：

```text
04-栏目协议/底层解码篇协议.md
```

写认知破局篇，必须存在：

```text
04-栏目协议/认知破局篇协议.md
```

### 6.3 阻断条件

如果没有栏目协议，阻断正文生成。

### 6.4 目的

防止栏目混乱。

## 7. guard-style-reverse

### 7.1 触发时机

当用户要求：

```text
写得像竞品
对标竞品
竞品质量
学习这个样文
```

### 7.2 检查内容

检查是否存在对应样文反推记录。

### 7.3 阻断条件

如果没有样文反推记录，禁止直接仿写。

提示：

```text
缺少样文反推，不能直接要求竞品质量。
请先执行样文深反推。
```

## 8. guard-source-evidence

### 8.1 触发时机

当写栏目文章或审查文章时触发。

### 8.2 检查内容

检查文章对应篇章是否存在原文拆解文件。

### 8.3 阻断条件

如果没有原文拆解，禁止成稿。

### 8.4 目的

防止文章变成空泛鸡汤或脱离原文的发挥。

## 9. pre-package

### 9.1 触发时机

当系统准备写入：

```text
08-项目打包/
```

### 9.2 检查内容

必须存在：

```text
06-质量审查/
```

并且目标文章必须有审查记录。

### 9.3 阻断条件

未审查文章不得进入打包。

## 10. pre-compact

### 10.1 触发时机

上下文压缩前。

### 10.2 功能

保存当前进度摘要到：

```text
追踪/上下文.md
```

摘要包含：

- 当前书籍。
- 当前栏目。
- 当前篇章。
- 已完成文件。
- 正在处理的任务。
- 下一步。

## 11. post-compact

### 11.1 触发时机

上下文压缩后。

### 11.2 功能

提示优先读取：

```text
追踪/上下文.md
```

防止上下文恢复后重新跑偏。

## 12. validate-commit

### 12.1 触发时机

git commit 前。

### 12.2 检查内容

检查是否存在明显问题：

- 有文章成稿但无栏目协议。
- 有文章成稿但无原文拆解。
- 有打包文件但无审查报告。
- 有新书项目但无书籍定位。

### 12.3 处理方式

第一阶段只警告，不阻断。

等流程稳定后，再决定哪些规则需要阻断。

## 13. 毛选样板特殊守卫

《毛选》样板第一阶段增加两个规则。

### 13.1 底层解码篇守卫

写底层解码篇前，必须存在：

```text
04-栏目协议/底层解码篇协议.md
02-原文拆解/对应篇章.md
01-样文反推/底层解码篇对应样文.md
```

### 13.2 认知破局篇守卫

写认知破局篇前，必须存在：

```text
04-栏目协议/认知破局篇协议.md
02-原文拆解/对应篇章.md
01-样文反推/认知破局篇对应样文.md
03-模型提炼/现实迁移/对应篇章.md
```

## 14. 第一阶段实现优先级

第一批必须实现：

1. session-start。
2. pre-write-column。
3. guard-column-protocol。
4. guard-source-evidence。
5. pre-compact。
6. post-compact。

第二批再实现：

1. guard-style-reverse。
2. pre-package。
3. validate-commit。
4. session-end。

## 15. 核心结论

hook 不负责让文章变好。

hook 负责让系统不要跳步。

文章质量靠：

```text
样文反推
-> 原文拆解
-> 栏目协议
-> 成稿
-> 审查
```

hook 的职责是确保这条链不断。
