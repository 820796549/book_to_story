# skill 路由设计

## 1. 设计目标

skill 路由的目标是让用户像使用 oh-story 一样使用拆书写作系统。

用户不需要每次重新解释完整流程，只需要说清当前任务，系统自动进入对应阶段。

主入口为：

```text
$book
```

专业入口为：

```text
$book-setup
$book-architect
$book-reverse-style
$book-decode-source
$book-extract-model
$book-map-application
$book-write-column
$book-style-polish
$book-review
$book-package
```

## 2. 主入口：$book

`$book` 是总路由。

它不直接完成所有任务，而是判断用户当前意图，并分发到对应 skill。

### 2.1 可识别意图

| 用户意图 | 路由到 |
|---|---|
| 初始化拆书项目 | `$book-setup` |
| 定位一本书 | `$book-architect` |
| 设计栏目 | `$book-architect` |
| 反推竞品/样文 | `$book-reverse-style` |
| 拆原文篇章 | `$book-decode-source` |
| 提炼模型/推理链 | `$book-extract-model` |
| 映射现实应用 | `$book-map-application` |
| 写栏目文章 | `$book-write-column` |
| 润色文章 | `$book-style-polish` |
| 审查文章 | `$book-review` |
| 打包项目 | `$book-package` |

### 2.2 模糊意图处理

当用户只说“帮我拆这本书”时，不能直接写正文。

`$book` 必须先判断当前项目是否已有：

1. 书籍定位。
2. 目标用户。
3. 栏目体系。
4. 样文反推。
5. 原文材料。

如果缺少定位，路由到 `$book-architect`。

如果缺少原文，提示用户提供原文路径或材料。

如果缺少栏目协议，不允许进入正文写作。

## 3. $book-setup

### 3.1 职责

初始化一个拆书写作项目。

### 3.2 输入

- 项目名称。
- 书名。
- 原文路径。
- 是否有样文。
- 是否有竞品。

### 3.3 输出

标准目录：

```text
00-项目定义/
01-样文反推/
02-原文拆解/
03-模型提炼/
04-栏目协议/
05-文章生产/
06-质量审查/
07-方法库/
08-项目打包/
追踪/
```

### 3.4 禁止

不得在初始化阶段直接生成正文。

## 4. $book-architect

### 4.1 职责

定义一本书为什么值得拆、给谁读、适合写成什么栏目。

### 4.2 输入

- 书名。
- 作者。
- 原文类型。
- 用户目标。
- 样文情况。

### 4.3 输出

```text
00-项目定义/书籍定位.md
00-项目定义/目标用户.md
00-项目定义/栏目体系.md
00-项目定义/使用路径.md
00-项目定义/内容地图.md
```

### 4.4 禁止

不得默认套用其他书的栏目。

## 5. $book-reverse-style

### 5.1 职责

反推竞品或高质量样文，把文章写法拆成可复用方法。

### 5.2 输入

- 样文路径。
- 样文所属栏目。
- 对应原文路径。
- 当前书籍定位。

### 5.3 输出

```text
01-样文反推/反推记录/*.md
07-方法库/标题公式.md
07-方法库/开头钩子.md
07-方法库/痛点入口.md
07-方法库/推理链模板.md
07-方法库/爽点句.md
07-方法库/结尾套路.md
07-方法库/禁止写法.md
```

### 5.4 反推字段

必须包含：

1. 文章一句话定位。
2. 标题机制。
3. 开头钩子。
4. 逐段功能。
5. 痛点设计。
6. 原文依据使用方式。
7. 推理链。
8. 爽点句类型。
9. 现实迁移方式。
10. 结尾套路。
11. 可入库方法。
12. 禁止入库内容。

## 6. $book-decode-source

### 6.1 职责

拆原文，不写文章。

### 6.2 输入

- 原文路径。
- 当前栏目目标。
- 书籍定位。

### 6.3 输出

```text
02-原文拆解/*.md
```

### 6.4 拆解字段

必须包含：

1. 篇章背景。
2. 核心问题。
3. 关键事实。
4. 作者判断。
5. 推理链。
6. 关键概念。
7. 可迁移模型。
8. 可引用原文。
9. 不应过度发挥的边界。

## 7. $book-extract-model

### 7.1 职责

从原文拆解中提炼模型、结构和方法。

### 7.2 输出

```text
03-模型提炼/*.md
```

模型可以是：

- 判断模型。
- 决策模型。
- 行动模型。
- 认知模型。
- 关系模型。
- 机制模型。

## 8. $book-map-application

### 8.1 职责

把书中的模型映射到现代用户场景。

### 8.2 输出

```text
03-模型提炼/现实迁移/*.md
```

### 8.3 禁止

不得脱离原文随意鸡汤化。

## 9. $book-write-column

### 9.1 职责

写用户可见栏目文章。

### 9.2 写作前置条件

必须存在：

1. 书籍定位。
2. 栏目协议。
3. 样文反推。
4. 原文拆解。
5. 模型提炼。

缺任一项，不得直接写正文。

### 9.3 输出

```text
05-文章生产/成稿/*.md
```

## 10. $book-style-polish

### 10.1 职责

在不改变论点和依据的前提下提升文章表达。

### 10.2 可改

- 标题。
- 开头。
- 段落节奏。
- 爽点句。
- 转场。
- 结尾。

### 10.3 不可改

- 原文事实。
- 核心判断。
- 栏目边界。
- 已确认的推理链。

## 11. $book-review

### 11.1 职责

审查文章是否合格。

### 11.2 审查维度

1. 是否符合书籍定位。
2. 是否符合栏目协议。
3. 是否有原文依据。
4. 是否推理清楚。
5. 是否有现实价值。
6. 是否有文章读感。
7. 是否存在竞品照搬。
8. 是否存在过度发挥。

### 11.3 输出

```text
06-质量审查/*.md
```

## 12. $book-package

### 12.1 职责

把已经通过审查的内容打包成可交付资产。

### 12.2 第一阶段限制

第一阶段只做文件整理，不做复杂产品化。

## 13. 路由原则

系统必须遵守以下路由原则：

1. 定位不清，不能写正文。
2. 栏目不清，不能写正文。
3. 没有原文拆解，不能写正文。
4. 没有样文反推，不能仿写竞品质量。
5. 没有审查，不能进入批量生产。
6. 用户要求太大时，拆成单文件任务。

## 14. 毛选第一阶段默认路由

当当前书籍项目是《毛选》时，优先路由为：

```text
书籍定位
-> 底层解码篇样文反推
-> 认知破局篇样文反推
-> 原文拆解
-> 栏目协议
-> 单篇试写
-> 质量审查
```

不得直接进入全卷批量生产。
