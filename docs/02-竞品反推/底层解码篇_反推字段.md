# 底层解码篇：反推字段

## 1. 栏目定位

底层解码篇不是读后感，也不是现代应用文。

它负责拆：

```text
在具体历史处境中，
作者如何识别问题、排除误判、建立判断链、找到破局点。
```

用于《毛选》时，它的核心任务是：

```text
还原教员在信息不全、资源不足、强敌环伺的条件下，
如何做局势判断和行动选择。
```

## 2. 反推目标

反推底层解码篇时，不是看文章“讲了哪些历史”，而是看它如何把历史材料写成决策算法。

反推文件必须能回答：

1. 这篇文章拆的是哪个历史困局？
2. 当时主流误判是什么？
3. 教员的判断链如何展开？
4. 哪个判断是真正破局点？
5. 文章如何让读者产生“原来问题在这里”的爽感？

## 3. 必填字段

### 3.1 样文信息

填写：

```text
样文标题：
对应原文：
所属卷/篇：
栏目：
样文路径：
原文路径：
反推日期：
```

### 3.2 一句话定位

格式：

```text
这篇底层解码用 [历史处境]，
拆出 [核心误判]，
还原 [推理链]，
最终交付 [破局判断]。
```

### 3.3 历史处境

必须拆出文章如何交代历史局。

关注：

- 时间点。
- 主要矛盾。
- 各方力量。
- 资源限制。
- 危险信号。
- 为什么这篇原文必须在此时出现。

判断标准：

```text
历史背景不是资料堆砌，
而是为后面的判断提供压力场。
```

### 3.4 当时误判

必须拆出文章指出了哪些错误判断。

常见误判：

- 看错主要敌人。
- 看错主要朋友。
- 高估表面力量。
- 低估潜在力量。
- 把口号当现实。
- 把局部经验当全局规律。
- 把道德态度当政治判断。

每个误判要写清：

```text
误判是什么：
为什么会这样误判：
这个误判会导致什么后果：
文章如何纠正它：
```

### 3.5 核心问题

拆出文章围绕什么问题展开。

格式：

```text
表层问题：
深层问题：
真正问题：
```

例如《毛选》第一篇不是只问“社会有哪些阶级”，而是问：

```text
谁是我们的敌人？
谁是我们的朋友？
革命为什么过去失败？
```

### 3.6 推理链

底层解码篇必须拆推理链。

建议格式：

```text
处境压力
-> 常见误判
-> 原文问题
-> 判断依据
-> 分类/模型
-> 行动结论
-> 历史验证
```

如果推理链只剩“作者认为”，反推不合格。

### 3.7 分类模型

如果样文把原文拆成分类体系，必须拆分类逻辑。

填写：

```text
分类依据：
分类对象：
每类特征：
每类态度：
每类处理方式：
```

用于《毛选》时，尤其要关注：

```text
经济地位
-> 政治态度
-> 革命关系
-> 策略安排
```

### 3.8 破局点

拆出文章认为真正的破局点是什么。

破局点不是一句漂亮话，而是一个判断转向。

格式：

```text
破局前的错误路线：
破局后的判断路线：
关键转向：
为什么这个转向有效：
```

### 3.9 历史验证

底层解码篇通常会用后续历史结果证明原文判断。

必须拆出：

- 文章用了哪个后续事件。
- 这个事件验证了什么判断。
- 这个验证是否合理。
- 是否存在过度倒推。

### 3.10 标题机制

拆标题如何把历史文章变成读者想看的问题。

关注：

- 是否提出失败原因。
- 是否制造悬念。
- 是否强调“底层”“洞穿”“破局”等词。
- 是否把历史问题转成普遍问题。

必须沉淀为标题公式，而不是复制标题。

### 3.11 开头结构

拆开头如何进入历史局。

常见结构：

```text
时间点
-> 局势压力
-> 多方误判
-> 原文登场
-> 核心问题
```

开头合格标准：

```text
读者能感觉到这篇文章不是凭空写的，
而是在一个必须做判断的关口出现的。
```

### 3.12 段落功能表

必须逐段拆。

表格：

| 段落/小节 | 表面内容 | 实际功能 | 服务哪条推理链 | 爽点/压力点 | 可复用写法 |
|---|---|---|---|---|---|

### 3.13 爽点句类型

底层解码篇的爽点通常不是情绪宣泄，而是判断穿透。

常见类型：

- 把失败原因一刀切开。
- 把复杂局势归到一个判断模型。
- 把表面口号拆成利益位置。
- 把历史结果反过来验证早期判断。
- 把“看人”改成“看结构”。

每类都要写：

```text
功能：
效果：
可复用句式：
禁用边界：
```

### 3.14 原文依据使用方式

拆样文如何调用原文。

关注：

- 是否直接引用。
- 是否转述观点。
- 是否提炼结构。
- 是否借原文做历史验证。
- 是否超出原文。

底层解码篇必须能回到原文。

### 3.15 禁止写法

必须记录这篇样文提醒我们不能怎么写。

常见禁止项：

- 把历史背景写成流水账。
- 把阶级分析写成标签贴人。
- 把教员神化成提前知道一切。
- 把复杂判断简化成一句口号。
- 过度现代化，把底层解码写成认知破局。

## 4. 输出模板

```text
# 底层解码篇深反推：样文标题

## 1. 样文信息
## 2. 一句话定位
## 3. 历史处境
## 4. 当时误判
## 5. 核心问题
## 6. 推理链
## 7. 分类模型
## 8. 破局点
## 9. 历史验证
## 10. 标题机制
## 11. 开头结构
## 12. 段落功能表
## 13. 爽点句类型
## 14. 原文依据使用方式
## 15. 可入库方法
## 16. 禁止写法
## 17. 对栏目协议的修正
## 18. 反推质量自检
```

## 5. 合格标准

底层解码篇反推合格，必须满足：

1. 能还原历史处境。
2. 能指出当时误判。
3. 能拆出推理链。
4. 能说明破局点。
5. 能区分历史材料和作者发挥。
6. 能沉淀标题、开头、推理、爽点、结尾方法。
7. 能反哺底层解码篇协议。

## 6. 不合格信号

出现以下情况，反推不合格：

1. 只总结文章内容。
2. 没有逐段功能表。
3. 没有拆误判。
4. 没有拆推理链。
5. 没有说明原文依据。
6. 只摘爽点句，不拆功能。
7. 把底层解码写成现代鸡汤。
