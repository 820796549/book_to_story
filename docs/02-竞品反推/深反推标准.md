# 深反推标准

## 1. 深反推的目的

深反推不是总结竞品文章。

深反推的目的是把一篇好文章拆成可复用的写作方法。

完成后的反推文件必须达到这个标准：

```text
不看竞品原文，
只看反推文件，
也能写出同类型文章。
```

如果反推文件只能说明“文章大概讲了什么”，就是失败。

## 2. 深反推和普通摘要的区别

普通摘要回答：

```text
这篇文章写了什么？
```

深反推回答：

```text
这篇文章为什么好读？
它如何抓住用户？
它如何组织材料？
它如何制造认知爽点？
它有哪些写法可以入库？
它有哪些表达不能照搬？
```

## 3. 必须拆的 14 个字段

每篇样文必须拆以下字段。

### 3.1 文章一句话定位

说明这篇文章实际完成了什么任务。

格式：

```text
这篇文章用 [材料/历史/原文]，
解决读者的 [痛点/困惑/误判]，
最终交付 [认知/模型/判断/行动启发]。
```

### 3.2 标题机制

拆标题如何吸引用户。

必须说明：

- 标题用了什么栏目标签。
- 标题提出了什么问题。
- 标题制造了什么悬念。
- 标题承诺了什么价值。
- 哪些词是爽点词。
- 哪些词不能照搬。

### 3.3 开头钩子

拆文章开头如何把读者拉进来。

必须说明：

- 开头从历史进入，还是从现代痛点进入。
- 第一段制造了什么紧张感。
- 开头有没有提出核心问题。
- 开头如何让读者觉得“这和我有关”。

### 3.4 逐段结构表

必须逐段拆，不得只写大概。

建议表格：

| 段落/小节 | 表面内容 | 实际功能 | 使用材料 | 情绪作用 | 可复用写法 |
|---|---|---|---|---|---|

重点不是复述内容，而是标出每段在文章中的功能。

### 3.5 痛点设计

拆文章命中了读者什么痛点。

痛点可以是：

- 看不懂局势。
- 不知道谁是朋友谁是敌人。
- 资源少，不知道怎么破局。
- 努力很多，但方向错了。
- 被强者包围，觉得没有机会。
- 现实中被复杂关系困住。

必须说明痛点是如何被文章逐步放大的。

### 3.6 误判设计

拆文章如何指出普通人的错误认知。

常见误判：

- 把口号当力量。
- 把热情当方法。
- 把强者当依靠。
- 把局部现象当全局。
- 把道德判断当现实判断。
- 把结果倒推成原因。

必须说明文章如何从误判转向正确模型。

### 3.7 原文依据使用方式

拆文章如何使用原文，而不是只看它引用了什么。

必须说明：

- 哪些地方直接使用原文观点。
- 哪些地方转述原文逻辑。
- 哪些地方借原文做现代迁移。
- 哪些地方没有原文依据，属于作者发挥。

要求：

```text
只允许少量短引用。
反推重点是使用方式，不是复制原文。
```

### 3.8 推理链

拆文章的核心推理结构。

格式：

```text
问题是什么
-> 常见误判是什么
-> 原文如何判断
-> 判断背后的依据是什么
-> 由此推出什么模型
-> 这个模型如何解释现实
```

如果推理链拆不出来，说明反推不合格。

### 3.9 爽点句类型

拆爽点句，不是摘抄爽点句。

必须说明每类爽点句的功能。

常见类型：

- 一句话翻案。
- 一句话点破。
- 一句话降维。
- 一句话定性。
- 一句话把历史和现实打通。
- 一句话把复杂问题变成简单判断。

每类至少说明：

```text
句子功能：
制造效果：
可复用句式：
禁用边界：
```

### 3.10 现实迁移方式

拆文章如何把书中内容迁移到现代读者。

必须说明：

- 迁移到了什么现代场景。
- 原文问题和现代问题有什么同构关系。
- 迁移是否过度。
- 迁移给读者带来什么启发。

### 3.11 结尾套路

拆文章如何收束。

常见结尾：

- 回到开头问题。
- 给出一句判断。
- 把历史结果作为验证。
- 把模型交还给读者。
- 留下一个现实问题。

必须说明结尾为什么有效。

### 3.12 可入库方法

把样文中可复用的写法沉淀为方法库条目。

可入库内容包括：

- 标题公式。
- 开头公式。
- 痛点公式。
- 推理链模板。
- 段落结构。
- 爽点句式。
- 现实迁移方法。
- 结尾模板。

入库时必须抽象成方法，不能直接复制表达。

### 3.13 禁止入库项

说明哪些东西不能学。

常见禁止项：

- 竞品的独特表达。
- 具体标题原句。
- 过度情绪化句子。
- 没有原文依据的发挥。
- 只适合某一篇文章的特殊结构。

### 3.14 对栏目协议的修正

反推结束后，必须回答：

```text
这篇样文说明当前栏目协议应该增加什么？
应该删除什么？
应该禁止什么？
```

深反推必须能反哺栏目协议。

## 4. 深反推输出格式

建议文件结构：

```text
# 样文深反推：文章标题

## 1. 样文信息
## 2. 文章一句话定位
## 3. 标题机制
## 4. 开头钩子
## 5. 逐段结构表
## 6. 痛点设计
## 7. 误判设计
## 8. 原文依据使用方式
## 9. 推理链
## 10. 爽点句类型
## 11. 现实迁移方式
## 12. 结尾套路
## 13. 可入库方法
## 14. 禁止入库项
## 15. 对栏目协议的修正
## 16. 反推质量自检
```

## 5. 质量自检

每篇深反推最后必须自检。

自检问题：

1. 是否逐段拆解，而不是概括？
2. 是否拆出标题机制？
3. 是否拆出开头钩子？
4. 是否拆出痛点和误判？
5. 是否拆出原文依据使用方式？
6. 是否拆出推理链？
7. 是否拆出爽点句类型？
8. 是否拆出现实迁移方式？
9. 是否拆出结尾套路？
10. 是否沉淀了可入库方法？
11. 是否明确了禁止入库项？
12. 是否能反哺栏目协议？

评分：

```text
90-100：可作为方法库来源
80-89：可作为反推记录，但需补强后入库
70-79：只能作为初稿
70 以下：重做
```

## 6. 对毛选样板的特殊要求

《毛选》样板的深反推必须额外关注：

### 6.1 底层解码篇

必须拆出：

```text
历史处境
-> 当时误判
-> 教员推理链
-> 破局点
-> 历史结果验证
```

### 6.2 认知破局篇

必须拆出：

```text
现代痛点
-> 错误认知
-> 毛选模型
-> 现实迁移
-> 爽点句
```

两个栏目不能混。

底层解码篇不能写成现代鸡汤。

认知破局篇不能写成历史资料总结。

## 7. 核心结论

深反推不是为了尊重竞品。

深反推是为了摆脱竞品。

只有把竞品写法拆成自己的方法库，后续写文章才不会每次都靠感觉。
