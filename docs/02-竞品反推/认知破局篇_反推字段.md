# 认知破局篇：反推字段

## 1. 栏目定位

认知破局篇不是历史解读，也不是单纯原文总结。

它负责拆：

```text
现代读者正在被什么困住，
这个困境背后的错误认知是什么，
书中提供了什么判断模型，
这个模型如何迁移到现实处境。
```

用于《毛选》时，它的核心任务是：

```text
借《毛选》的判断方式，
解决现代人在资源少、局势乱、关系复杂、行动受阻时的认知困境。
```

## 2. 反推目标

反推认知破局篇时，不是看文章“讲了哪些毛选知识”，而是看它如何让读者觉得：

```text
这说的不是历史，这是我现在的处境。
```

反推文件必须能回答：

1. 文章命中了什么现代痛点？
2. 读者原本的错误认知是什么？
3. 原文提供了什么模型？
4. 文章如何完成历史到现实的迁移？
5. 爽点句如何让读者产生认知翻转？

## 3. 必填字段

### 3.1 样文信息

填写：

```text
样文标题：
对应原文：
所属卷/篇：
栏目：
样文路径：
原文路径：
反推日期：
```

### 3.2 一句话定位

格式：

```text
这篇认知破局用 [原文模型]，
击穿读者在 [现代处境] 中的 [错误认知]，
最终交付 [现实判断/行动启发/认知转向]。
```

### 3.3 现代痛点

必须拆出文章从哪个现实痛点切入。

常见痛点：

- 努力很多但方向不对。
- 不知道该联合谁。
- 总想找强者依靠。
- 看不清局势中的真正变量。
- 被复杂关系消耗。
- 资源少但又想破局。
- 想行动但害怕失败。
- 把情绪当判断。

每个痛点要写清：

```text
读者处境：
读者感受：
读者常见反应：
这个反应为什么错：
```

### 3.4 错误认知

认知破局篇的核心是“破”。

必须拆出文章破掉了什么错误认知。

常见错误认知：

- 以为努力本身能改变处境。
- 以为站队强者就安全。
- 以为资源少就没有机会。
- 以为人际关系靠情分决定。
- 以为看清表态就等于看清立场。
- 以为失败是能力不足，而不是局势判断错误。

格式：

```text
错误认知：
它为什么有迷惑性：
它带来的后果：
文章如何推翻它：
```

### 3.5 毛选模型

必须拆出文章从原文中提炼了什么模型。

模型不是口号。

模型必须能说明：

```text
问题如何识别
变量如何排序
关系如何判断
行动如何选择
```

用于《毛选》时，常见模型包括：

- 敌友识别模型。
- 主要矛盾判断模型。
- 资源劣势破局模型。
- 群众基础模型。
- 调查研究模型。
- 实事求是模型。
- 组织路线模型。

### 3.6 历史材料使用方式

认知破局篇也要使用历史，但历史不是主体。

必须拆出：

- 文章用了哪些历史材料。
- 这些材料服务哪个现代痛点。
- 历史材料有没有过多。
- 历史是否压住了现实迁移。

判断标准：

```text
历史材料是模型来源，
现代处境是读者入口。
```

### 3.7 现实迁移链

认知破局篇必须拆出现实迁移链。

格式：

```text
原文问题
-> 原文判断模型
-> 现代同构处境
-> 读者错误反应
-> 新判断方式
-> 可执行启发
```

如果只有“古今都一样”，反推不合格。

### 3.8 爽点句类型

认知破局篇的爽点来自认知翻转。

常见类型：

- 把读者的痛苦重新定义。
- 把努力失败归因到判断错误。
- 把复杂人际关系变成结构判断。
- 把弱者处境转成破局入口。
- 把历史模型打到现代现实。

每类都要写：

```text
功能：
击中的痛点：
制造的认知翻转：
可复用句式：
禁用边界：
```

### 3.9 标题机制

拆标题如何击中现代读者。

关注：

- 是否提出一个现实困境。
- 是否承诺破局。
- 是否有反常识判断。
- 是否把《毛选》从历史书变成现实工具。

标题反推必须沉淀为公式。

不能复制竞品标题。

### 3.10 开头结构

认知破局篇的开头通常优先从现代痛点进入。

常见结构：

```text
现代困境
-> 普通人误判
-> 为什么越努力越错
-> 引入原文模型
-> 提出破局问题
```

也可以从历史进入，但必须快速转到现代读者。

### 3.11 段落功能表

必须逐段拆。

表格：

| 段落/小节 | 表面内容 | 实际功能 | 对应痛点 | 对应模型 | 爽点/转折 | 可复用写法 |
|---|---|---|---|---|---|---|

### 3.12 读者情绪曲线

认知破局篇必须拆读者情绪如何变化。

常见曲线：

```text
被说中
-> 发现自己误判
-> 看到新模型
-> 感到局势变清楚
-> 产生行动欲望
```

必须说明文章如何推动这条曲线。

### 3.13 结尾套路

认知破局篇结尾通常要把模型交还给读者。

常见结尾：

- 回到现代处境。
- 给出一句认知判断。
- 提醒读者下次遇到问题如何看。
- 把原文模型转成行动起点。
- 留下一个现实问题。

必须说明结尾是否有力量，是否过度鸡汤。

### 3.14 原文依据和发挥边界

必须区分：

- 哪些来自原文。
- 哪些来自样文作者的现代解释。
- 哪些是合理迁移。
- 哪些可能过度发挥。

认知破局篇最容易脱离原文。

反推时必须把边界写清。

### 3.15 禁止写法

必须记录不能怎么写。

常见禁止项：

- 只讲现代焦虑，不回到原文。
- 只讲历史，不解决现代痛点。
- 把毛选模型写成万能答案。
- 把复杂现实简化成情绪鸡汤。
- 为了爽点制造不准确判断。
- 把底层解码篇的历史结构硬塞进认知破局篇。

## 4. 输出模板

```text
# 认知破局篇深反推：样文标题

## 1. 样文信息
## 2. 一句话定位
## 3. 现代痛点
## 4. 错误认知
## 5. 毛选模型
## 6. 历史材料使用方式
## 7. 现实迁移链
## 8. 爽点句类型
## 9. 标题机制
## 10. 开头结构
## 11. 段落功能表
## 12. 读者情绪曲线
## 13. 结尾套路
## 14. 原文依据和发挥边界
## 15. 可入库方法
## 16. 禁止写法
## 17. 对栏目协议的修正
## 18. 反推质量自检
```

## 5. 合格标准

认知破局篇反推合格，必须满足：

1. 能明确现代痛点。
2. 能指出错误认知。
3. 能拆出原文模型。
4. 能说明历史如何迁移到现实。
5. 能拆出读者情绪曲线。
6. 能拆出爽点句类型。
7. 能区分原文依据和现代发挥。
8. 能反哺认知破局篇协议。

## 6. 不合格信号

出现以下情况，反推不合格：

1. 只总结文章内容。
2. 没有现代痛点。
3. 没有错误认知。
4. 没有现实迁移链。
5. 没有读者情绪曲线。
6. 爽点句只摘抄不分析。
7. 把文章写成历史资料总结。
8. 把文章写成空泛鸡汤。
