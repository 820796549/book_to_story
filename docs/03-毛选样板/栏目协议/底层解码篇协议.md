# 毛选栏目协议：底层解码篇

## 1. 栏目定位

底层解码篇负责把《毛选》中的篇章，拆成历史处境中的决策算法。

它不是讲“这篇文章说了什么”，而是讲：

```text
在当时那个局势下，
为什么必须这么判断，
为什么别人会误判，
教员如何从混乱局面中找到破局点。
```

一句话定位：

```text
底层解码篇 = 历史处境 + 当时误判 + 推理链 + 破局点。
```

## 2. 目标读者

底层解码篇服务的读者不是单纯想读历史的人。

它服务的是：

- 想理解《毛选》为什么厉害的人。
- 想学习复杂局势判断的人。
- 想知道弱势条件下如何找到突破口的人。
- 想从历史决策中提炼现实判断能力的人。
- 看过普通解读但觉得不够透的人。

## 3. 文章必须回答的问题

每篇底层解码必须回答四个问题：

1. 当时到底卡在什么局里？
2. 大多数人为什么看错？
3. 教员的判断链是什么？
4. 真正的破局点在哪里？

如果这四个问题没有回答清楚，文章不合格。

## 4. 标准结构

建议结构：

```text
标题
-> 历史开场
-> 核心问题
-> 当时误判
-> 原文推理链
-> 关键分类/模型
-> 破局判断
-> 历史验证
-> 收束
```

结构可以根据篇章微调，但不能缺少：

- 历史处境。
- 误判。
- 推理链。
- 破局点。

## 5. 历史开场要求

历史开场不是背景资料堆砌。

它必须制造一个“不得不判断”的压力场。

开场要交代：

1. 这篇文章出现在哪个历史节点。
2. 当时主要矛盾是什么。
3. 各方力量如何变化。
4. 如果判断错，会导致什么后果。
5. 为什么这篇文章在此时有必要出现。

不合格开场：

```text
某年某月发生了什么，然后又发生了什么。
```

合格开场：

```text
某个历史关口上，旧路线已经解释不了现实，
各方力量正在重新分化，
如果还按原来的眼光看局势，就会把朋友看成旁观者，把敌人看成依靠。
```

## 6. 误判要求

底层解码篇必须写误判。

误判是文章的张力来源。

常见误判：

- 把表面强者当成真正依靠。
- 把人数众多但沉默的群体当成背景。
- 把口号当立场。
- 把情绪当判断。
- 把局部经验当全局规律。
- 把道德好恶当现实分析。

每个误判建议写成：

```text
当时很多人以为：
但问题在于：
真正决定局势的是：
```

## 7. 推理链要求

底层解码篇必须有清楚推理链。

推荐结构：

```text
局势压力
-> 常见误判
-> 原文提出核心问题
-> 找到判断依据
-> 建立分类或模型
-> 推出行动方向
-> 后续历史验证
```

推理链不能写成：

```text
教员很厉害，所以他看对了。
```

必须写成：

```text
他依据什么变量，
排除了什么假象，
推出什么判断。
```

## 8. 破局点要求

破局点不是“结论正确”。

破局点是视角切换。

例如：

- 从看口号，转为看利益位置。
- 从看上层人物，转为看底层力量。
- 从看眼前强弱，转为看发展趋势。
- 从看单个事件，转为看结构性矛盾。
- 从被动等待，转为主动组织。

每篇必须明确写出：

```text
真正的破局点是：
```

## 9. 原文依据要求

底层解码篇必须有原文依据。

可以使用：

- 原文核心问题。
- 原文分类。
- 原文判断。
- 原文关键句。
- 原文论证顺序。

但不能大段照抄原文。

文章应该重点说明：

```text
原文如何组织判断。
```

而不是：

```text
原文有哪些句子。
```

## 10. 表达风格

底层解码篇的表达要有穿透感，但不能玄。

推荐风格：

- 判断明确。
- 句子有力。
- 少用空泛形容词。
- 多写结构和因果。
- 用历史细节承托判断。
- 关键句要能让读者感到“原来问题在这里”。

避免：

- 神化教员。
- 空喊伟大。
- 泛泛讲历史。
- 现代鸡汤化。
- 大量堆砌背景。
- 把复杂判断写成口号。

## 11. 爽点句要求

底层解码篇的爽点句来自判断穿透。

常见句式：

```text
真正的问题不是 X，而是 Y。
```

```text
他不是在判断谁更强，而是在判断谁代表未来的力量。
```

```text
一场斗争最危险的地方，不是敌人强，而是你把朋友看错。
```

```text
所谓破局，第一步不是行动，而是重新画出局势中的力量地图。
```

使用要求：

- 爽点句必须服务推理链。
- 不能为了爽而制造不准确判断。
- 每篇文章不需要句句都炸，关键处有力量即可。

## 12. 结尾要求

结尾要完成三件事：

1. 回到本篇核心判断。
2. 点明这篇原文在后续历史中的意义。
3. 留给读者一个可迁移的判断方式。

结尾不能突然变成现代鸡汤。

可用结构：

```text
这篇文章真正厉害的地方，
不是列出了一张分类表，
而是把革命从热情拉回了判断。

先看清谁在局中，谁会动，谁能联合，谁必须打击，
后面的路线才有可能展开。
```

## 13. 禁止写法

底层解码篇禁止：

1. 写成普通历史介绍。
2. 写成原文摘要。
3. 写成现代职场启发。
4. 写成纯情绪爽文。
5. 神化作者，忽略具体推理。
6. 强行套用固定金句。
7. 没有原文依据就下判断。
8. 把认知破局篇内容混进来。

## 14. 成稿检查

写完后自检：

1. 有没有历史处境？
2. 有没有当时误判？
3. 有没有核心问题？
4. 有没有推理链？
5. 有没有破局点？
6. 有没有原文依据？
7. 有没有历史验证？
8. 有没有栏目边界？
9. 有没有读者能记住的判断句？
10. 有没有过度现代化？

少于 8 项通过，不得进入审查。
