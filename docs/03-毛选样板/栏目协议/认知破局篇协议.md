# 毛选栏目协议：认知破局篇

## 1. 栏目定位

认知破局篇负责把《毛选》中的判断模型，迁移到现代人的现实处境。

它不是讲历史本身，而是讲：

```text
现代人为什么会被某个困局困住，
这个困局背后的错误认知是什么，
《毛选》提供了什么判断模型，
这个模型如何帮助读者重新看问题。
```

一句话定位：

```text
认知破局篇 = 现代痛点 + 错误认知 + 毛选模型 + 现实迁移 + 爽点句。
```

## 2. 目标读者

认知破局篇服务的读者是现实中有困惑的人。

他们可能：

- 资源少，但想找突破口。
- 努力很多，但总是方向错。
- 看不清谁能合作，谁在消耗自己。
- 总想依靠强者，却反复失望。
- 面对复杂局势，不知道抓哪个变量。
- 学了很多道理，却无法转成判断。

他们读这类文章，不是为了补历史知识，而是为了获得现实判断力。

## 3. 文章必须回答的问题

每篇认知破局必须回答五个问题：

1. 现代读者被什么困住？
2. 读者原本的错误认知是什么？
3. 《毛选》中的对应模型是什么？
4. 这个模型如何迁移到现实？
5. 读者下次遇到类似问题应该怎么重新判断？

如果只回答历史问题，文章不合格。

## 4. 标准结构

建议结构：

```text
标题
-> 现代痛点开场
-> 点出错误认知
-> 引入毛选原文/历史处境
-> 提炼判断模型
-> 映射现代处境
-> 给出认知转向
-> 爽点句强化
-> 收束到现实判断
```

结构可以微调，但不能缺少：

- 现代痛点。
- 错误认知。
- 毛选模型。
- 现实迁移。

## 5. 现代痛点要求

认知破局篇必须从读者的现实困境切入，或者快速落回现实困境。

痛点要具体，不能空泛。

不合格痛点：

```text
很多人都很迷茫。
```

合格痛点：

```text
很多人不是不努力，而是从一开始就把局势看反了：
他拼命讨好看起来最强的人，却忽略了真正能和他一起改变局面的人。
```

痛点必须让读者感觉：

```text
这说的是我现在遇到的问题。
```

## 6. 错误认知要求

认知破局篇必须先“破”。

常见错误认知：

- 以为努力能解决方向错误。
- 以为强者就是最值得依靠的人。
- 以为资源少就没有机会。
- 以为关系靠情分，而不是靠结构位置。
- 以为表态等于立场。
- 以为失败是能力问题，而不是判断模型问题。

建议写法：

```text
你以为问题是 X。
但真正的问题是 Y。
```

或者：

```text
很多人输，不是输在不努力，
而是输在一开始就把局里的力量看错了。
```

## 7. 毛选模型要求

认知破局篇不能只借《毛选》做背书。

必须提炼出明确模型。

模型必须说明：

1. 如何识别问题。
2. 如何排序变量。
3. 如何判断关系。
4. 如何找到行动方向。

常见模型：

- 敌友识别模型。
- 主要矛盾模型。
- 调查研究模型。
- 群众路线模型。
- 弱势破局模型。
- 组织力量模型。
- 实事求是模型。

模型不能写成口号。

不合格：

```text
要实事求是。
```

合格：

```text
实事求是的关键不是态度诚恳，
而是先把真实变量找出来：
谁受损，谁获益，谁有行动意愿，谁只是口头支持。
```

## 8. 现实迁移要求

现实迁移必须满足三个条件：

1. 现代处境和原文问题有结构相似性。
2. 迁移后能帮助读者重新判断。
3. 不把历史结论硬套到现代问题。

推荐迁移链：

```text
原文问题
-> 原文模型
-> 现代同构处境
-> 读者常见误判
-> 新判断方式
-> 可执行启发
```

禁止：

```text
古代/历史是这样，所以你现在也应该这样。
```

必须写清楚“为什么可以迁移”。

## 9. 历史材料使用要求

认知破局篇可以写历史，但历史不能压住现实。

历史材料的作用是：

- 提供模型来源。
- 增加判断重量。
- 说明这个模型不是凭空发明。

历史材料不应该变成：

- 长篇背景介绍。
- 原文摘要。
- 底层解码篇式推演。

判断标准：

```text
读者读完后记住的是一个现实判断方式，
不是一段历史资料。
```

## 10. 表达风格

认知破局篇要更贴近现代读者。

推荐风格：

- 开头直接。
- 问题具体。
- 判断有力。
- 多用“你以为/真正”的认知翻转。
- 用短句制造击穿感。
- 用模型承接情绪，不能只煽动情绪。

避免：

- 空泛鸡汤。
- 情绪过猛但没有依据。
- 把《毛选》写成万能答案。
- 把复杂现实一刀切。
- 只顾现代痛点，不回到原文模型。

## 11. 爽点句要求

认知破局篇的爽点来自认知翻转。

常见句式：

```text
你不是没有机会，你是把机会看错了地方。
```

```text
很多人所谓的努力，只是在错误地图上越跑越远。
```

```text
弱者破局的第一步，不是找到贵人，而是找到和自己处境相连的人。
```

```text
真正的清醒，不是看谁说得好听，而是看谁的位置决定了他会怎么动。
```

使用要求：

- 爽点句必须来自前文模型。
- 不能无依据制造狠话。
- 每篇文章要有关键记忆点。

## 12. 结尾要求

结尾要把模型还给读者。

推荐结构：

```text
所以这篇文章给我们的不是一个历史知识点，
而是一种看局势的方法。

下次你再面对复杂关系时，
不要先问谁看起来强，
先问谁和你的处境相连，谁有行动意愿，谁只是口头支持。
```

结尾不能：

- 突然喊口号。
- 空泛鼓励。
- 脱离原文。
- 给出不负责任的现实建议。

## 13. 与底层解码篇的边界

认知破局篇和底层解码篇必须区分。

底层解码篇重点是：

```text
当时历史局势中，教员如何判断。
```

认知破局篇重点是：

```text
现代现实处境中，读者如何借这个模型重新判断。
```

同一篇原文可以写两个栏目，但文章重心不同。

如果文章大部分篇幅都在讲历史处境，就是底层解码。

如果文章大部分篇幅都在讲现代痛点，但没有原文模型，就是鸡汤。

认知破局篇必须同时有：

```text
现代痛点 + 原文模型 + 现实迁移
```

## 14. 禁止写法

认知破局篇禁止：

1. 写成原文摘要。
2. 写成历史资料。
3. 写成现代鸡汤。
4. 写成成功学。
5. 写成万能方法论。
6. 脱离原文硬套现实。
7. 为了爽点牺牲准确性。
8. 和底层解码篇边界混乱。

## 15. 成稿检查

写完后自检：

1. 有没有具体现代痛点？
2. 有没有明确错误认知？
3. 有没有毛选原文模型？
4. 有没有现实迁移链？
5. 有没有原文依据？
6. 有没有读者情绪曲线？
7. 有没有关键爽点句？
8. 有没有避免鸡汤化？
9. 有没有和底层解码篇区分开？
10. 有没有给读者留下可复用判断方式？

少于 8 项通过，不得进入审查。
